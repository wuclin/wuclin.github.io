有序集合

LinkedList

用链表实现，java中都是双向链表，隐藏了在删除元素时调整指针的繁琐操作。

ArrayList

关于动态数组，有ArrayList和Vector。

一般会使用ArrayList，因为Vector的所有方法都是同步的，虽然可以安全的从两个线程访问同一个Vector对象，但是如果从一个线程中访问Vector（这种情况比较常见），会浪费时间在同步操作上。因此在不需要同步的时候使用ArrayList，需要同步时使用Vector。

Set接口

等同于Collection，其中的add方法不允许增加重复的元素

散列表

如果自己实现自己的类，要自己实现hashCode方法，实现要与equals方法兼容，即如果a.equals(b)为true，a与b要有相同的散列码。

Java中使用链表数组实现，使用哈希散列法。

Java集合类提供HashSet（散列集），使用场景：不关心集合中元素的顺序；TreeSet（树集），有序集合，在进行遍历时自动按照排序后的顺序呈现。

1.java环境

1.1 JVM, JDR,JRE

JVM：Java虚拟机，是运行Java字节码（.class后缀）的虚拟机，

JDK : 编写Java程序使用软件，Java开发工具包（这也是为什么一般都是下载JDK的原因）

JRE : 运行Java程序使用软件，Java运行时环境（只包含虚拟机）

SE : 标准版，桌面或简单服务器

EE : 企业版，复杂服务器

1.2 Java，编译和解释并存

编译型语言是指编译器针对特定的操作系统将源代码一次性翻译成可被该平台执行的机器码；解释型语言是指解释器对源程序逐行解释成特定平台的机器码并立即执行。比如，你想阅读一本英文名著，你可以找一个英文翻译人员帮助你阅读， 有两种选择方式，你可以先等翻译人员将全本的英文名著（也就是源码）都翻译成汉语，再去阅读，也可以让翻译人员翻译一段，你在旁边阅读一段，慢慢把书读完。

Java程序要先经过编译，生成字节码（*.class文件），这种字节码再由Java解释器来解释执行。因此，可以认为Java编译与解释并存。

git命令

Posted on 2020-03-30

常用命令

  • 查看当前分支
    git checkout

  • 查看当前项目所有的分支
    git branch -a

  • 切换分支
    git checkout 目标分支

  • 查看本地分支
    git branch

  • 创建分支
    git branch 分支名称

  • 删除本地分支
    git branch -d 分支名称

  • 推送本地分支到远程仓库
    git push <remotename> <branchname>
    git push 有两个参数：一个是远程仓库名字；一个是分支名字
    git push 是 git push origin master的简化
    默认的[remote, baranch] = [origin, master]

    a.远程已有remote_branch分支并且已经关联本地分支local_branch且本地已经切换到local_branch

    git push

    b.远程已有remote_branch分支但未关联本地分支local_branch且本地已经切换到local_branch

    git push -u origin/remote_branch

    c.远程没有remote_branch分支并，本地已经切换到local_branch

    git push origin local_branch:remote_branch
