Spring/SpringBoot常用注解

Posted on 2021-01-20 Edited on 2021-02-15 In Spring

Spring注解

Bean管理注解：

@Component（组件不好归类时，使用这个注解），@Component的三个衍生注解：

  • @Controller：web层
  • @Service：业务层
  • @Repository：持久层，在daoImpl类上面注解

属性注入注解：

使用注解注入的方式,可以不用提供set方法

@Autowired：

自动导入对象到类中，被注入进的类同样要被 Spring 容器管理比如：Service 类注入到 Controller 类中。

@RequestMapping

@ResponseBody （异步请求，通常用于ajax调用，一般时返回json或xml，而不是html页面）

@RestController 相当于 @Controller和@ResponseBody

参数处理注解：

@PathVarible：处理request uri 部分的注解

@RequestParam，@RequestBody：处理request header部分的注解

@RequestParam:

可用来处理简单的绑定

@RequestBody:

该注解常用来处理Content-Type: 不是application/x-www-form-urlencoded编码的内容，例如application/json, application/xml等；

@SessionAttribute，@ModelAttribute：处理attribute类型的注解

@SessionAttributes:

该注解用来绑定HttpSession中的attribute对象的值，便于在方法中的参数里使用。

该注解有value、types两个属性，可以通过名字和类型指定要使用的attribute 对象；

SpringBoot注解

@SrpingBootApplication

可以把 @SpringBootApplication看作是 @Configuration、@EnableAutoConfiguration、@ComponentScan 注解的集合。

这个注解是 Spring Boot 项目的基石，创建 SpringBoot 项目之后会默认在主类加上。

根据 SpringBoot 官网，这三个注解的作用分别是：

  • @EnableAutoConfiguration：启用 SpringBoot 的自动配置机制
  • @ComponentScan： 扫描被@Component (@Service,@Controller)注解的 bean，注解默认会扫描该类所在的包下所有的类。
  • @Configuration：允许在 Spring 上下文中注册额外的 bean 或导入其他配置类
