Spring/SpringBoot常用注解

Posted on 2021-01-20 Edited on 2021-01-26 In Spring

1.@SrpingBootApplication

可以把 @SpringBootApplication看作是 @Configuration、@EnableAutoConfiguration、@ComponentScan 注解的集合。

这个注解是 Spring Boot 项目的基石，创建 SpringBoot 项目之后会默认在主类加上。

根据 SpringBoot 官网，这三个注解的作用分别是：

  • @EnableAutoConfiguration：启用 SpringBoot 的自动配置机制
  • @ComponentScan： 扫描被@Component (@Service,@Controller)注解的 bean，注解默认会扫描该类所在的包下所有的类。
  • @Configuration：允许在 Spring 上下文中注册额外的 bean 或导入其他配置类

2.Spring Bean相关

第一类

@Controller

标注控制层组件，标记在类上，表示为SpringMVC Controller对象，会检测该方法是否使用了@RequestMapping注解

@RestController

相当于 @Controller和@ResponseBody

@Component

组件不好归类时，使用这个注解

@Repository

用于注解dao层，在daoImpl类上面注解

@Service

用于标注业务层组件

第二类

@ResponseBody

异步请求，通常用于ajax调用，一般时返回json或xml，而不是html页面

@RequestMapping

用于处理请求地址映射的注解，可用在类或方法上

@Autowired

可对类成员变量，方法，构造函数标注，完成自动装配的工作，通过@Autowired消除set，get方法

@PathVariable

@RequestParam

  1. @Autowired

    自动导入对象到类中，被注入进的类同样要被 Spring 容器管理比如：Service 类注入到 Controller 类中。

  2. @Component,@Repository,@Service, @Controller

  3. @RestControler

  4. @Scope
