List，Set，Map的区别

  • List： 存储的元素是有序的、可重复的。
  • Set: 存储的元素是无序的、不可重复的。
  • Map: 使用键值对（kye-value）存储，类似于数学上的函数 y=f(x)，“x”代表 key，”y”代表 value，Key 是无序的、不可重复的，value 是无序的、可重复的，每个键最多映射到一个值。

LinkedList

用链表实现，java中都是双向链表（JDK1.7取消了循环），隐藏了在删除元素时调整指针的繁琐操作。

ArrayList

关于动态数组，有ArrayList和Vector（底层为Object数组）。

一般会使用ArrayList，因为Vector的所有方法都是同步的，虽然可以安全的从两个线程访问同一个Vector对象，但是如果从一个线程中访问Vector（这种情况比较常见），会浪费时间在同步操作上。因此在不需要同步的时候使用ArrayList，需要同步时使用Vector（线程安全）。

关于RandomAccess接口

RandomAccess 接口只是标识，ArrayList 实现了 RandomAccess 接口， 而 LinkedList 没有实现。只有支持快速随机访问，才会实现该接口。

散列表

如果自己实现自己的类，要自己实现hashCode方法，实现要与equals方法兼容，即如果a.equals(b)为true，a与b要有相同的散列码。

Java中使用链表数组实现，使用哈希散列法。

Java集合类提供HashSet（散列集），使用场景：不关心集合中元素的顺序；TreeSet（树集），有序集合，在进行遍历时自动按照排序后的顺序呈现。

总结：

  • HashSet（无序，唯一）: 基于 HashMap 实现的，底层采用 HashMap 来保存元素
  • LinkedHashSet：LinkedHashSet 是 HashSet 的子类，并且其内部是通过 LinkedHashMap 来实现的。有点类似于我们之前说的 LinkedHashMap 其内部是基于 HashMap 实现一样，不过还是有一点点区别的
  • TreeSet（有序，唯一）： 红黑树(自平衡的排序二叉树)

无序性和不可重复性的含义是什么

1、什么是无序性？无序性不等于随机性 ，无序性是指存储的数据在底层数组中并非按照数组索引的顺序添加 ，而是根据数据的哈希值决定的。

2、什么是不可重复性？不可重复性是指添加的元素按照 equals()判断时 ，返回 false，需要同时重写 equals()方法和 HashCode()方法。

HashMap 和 HashSet 区别

如果你看过 HashSet 源码的话就应该知道：HashSet 底层就是基于 HashMap 实现的。（HashSet 的源码非常非常少，因为除了 clone()、writeObject()、readObject()是 HashSet 自己不得不实现之外，其他方法都是直接调用 HashMap 中的方法。

HashMap HashSet
实现了 Map 接口 实现 Set 接口
存储键值对 仅存储对象
调用 put()向 map 中添加元素 调用 add()方法向 Set 中添加元素
HashMap 使用键（Key）计算 hashcode HashSet 使用成员对象来计算 hashcode 值，对于两个对象来说 hashcode 可能相同，所以equals()方法用来判断对象的相等性

HashMap 和 TreeMap 区别

TreeMap 和HashMap 都继承自AbstractMap ，但是需要注意的是TreeMap它还实现了NavigableMap接口和SortedMap 接口。

实现 NavigableMap 接口让 TreeMap 有了对集合内元素的搜索的能力。

实现SortMap接口让 TreeMap 有了对集合中的元素根据键排序的能力。默认是按 key 的升序排序，不过我们也可以指定排序的比较器。

HashMap底层实现

使用拉链法，不过当链表长度大于阈值时，（将链表转换成红黑树前会判断，如果当前数组的长度小于 64，那么会选择先进行数组扩容，而不是转换为红黑树）时，将链表转化为红黑树，以减少搜索时间。

集合的选用

主要根据集合的特点来选用，比如我们需要根据键值获取到元素值时就选用 Map 接口下的集合，需要排序时选择 TreeMap,不需要排序时就选择 HashMap,需要保证线程安全就选用 ConcurrentHashMap。

当我们只需要存放元素值时，就选择实现Collection 接口的集合，需要保证元素唯一时选择实现 Set 接口的集合比如 TreeSet 或 HashSet，不需要就选择实现 List 接口的比如 ArrayList 或 LinkedList，然后再根据实现这些接口的集合的特点来选用。

为什么要使用集合

当我们需要保存一组类型相同的数据的时候，我们应该是用一个容器来保存，这个容器就是数组，但是，使用数组存储对象具有一定的弊端， 因为我们在实际开发中，存储的数据的类型是多种多样的，于是，就出现了“集合”，集合同样也是用来存储多个数据的。

数组的缺点是一旦声明之后，长度就不可变了；同时，声明数组时的数据类型也决定了该数组存储的数据的类型；而且，数组存储的数据是有序的、可重复的，特点单一。 但是集合提高了数据存储的灵活性，Java 集合不仅可以用来存储不同类型不同数量的对象，还可以保存具有映射关系的数据。
